1. Explique o que é o MySQL e cite ao menos três de suas vantagens.

    Mysql é um sistema de gerenciamento de banco de dados relacional e suas 3
    principais vantagens são: custo zero (open source), alta escalabilidade para 
    grandes volumes de dados e fácil integração com diversas linguagens de programação

2. Qual a diferença entre um banco de dados relacional e um banco de dados não relacional? Dê exemplos práticos de uso.

    Bancos de dados relacionais (SQL) organizam dados em tabelas rígidas com linhas e colunas,
    garantindo alta consistência (ACID). Não relacionais (NoSQL) usam modelos flexíveis 
    (documentos, chave-valor, grafos) para dados não estruturados, 
    oferecendo escalabilidade horizontal e alta velocidade.

    exemplo SQL: Sistemas Bancários/Financeiros: Exigem consistência absoluta para transações (ACID).

    exemplo NoSQL: Redes Sociais e Catálogos (MongoDB): Armazenar perfis de usuários com informações variadas (não estruturadas).

3. Cite e explique os principais componentes da arquitetura do MySQL.

    1. Conectores: Permitem que aplicações (Java, Python, PHP, etc.) se conectem ao MySQL.
    2. Gerenciador de Consultas (Query Processor): é responsável por interpretar comandos SQL, otimizar consultas
    executar operações no banco.
    3. Motores de Armazenamento (Storage Engines): definem como os dados são armazenados fisicamente.
    4. Buffers e Cache: guardam dados em memória para evitar acesso constante ao disco, melhorando desempenho.

4. Qual a função do motor de armazenamento do MySQL? Explique a diferença entre InnoDB e MyISAM.
    
    O motor de armazenamento do MySQL tem a função de definir como os dados serão armazenados, organizados e acessados dentro do banco de dados. 
    Ele também controla aspectos como indexação, transações e integridade dos dados. O InnoDB é o motor mais utilizado atualmente, pois suporta transações, 
    garante propriedades ACID, permite o uso de chaves estrangeiras e oferece maior segurança em operações críticas. 
    Já o MyISAM é um motor mais simples e rápido para leitura, porém não suporta transações nem integridade referencial, 
    o que o torna menos seguro para aplicações que exigem consistência de dados.

5. Identifique e explique a função de três ferramentas de interface gráfica utilizadas para o MySQL.

    MySQL Workbench, o DBeaver e o phpMyAdmin.
    O MySQL Workbench é a ferramenta oficial da Oracle, utilizada para modelagem de banco de dados, execução de consultas e administração do servidor.
    O DBeaver é uma ferramenta multiplataforma que suporta diversos tipos de bancos de dados, sendo bastante utilizada por desenvolvedores que trabalham com múltiplos SGBDs. 
    Já o phpMyAdmin é uma interface baseada em navegador, muito utilizada em ambientes web, permitindo gerenciar bancos de dados MySQL de forma prática sem a necessidade de instalação local de software cliente.

14. Suponha que você precise garantir que apenas usuários autenticados possam acessar um banco de dados MySQL. Que práticas de segurança você aplicaria?

    Para garantir acesso apenas a usuários autenticados:
        1.Criar usuários específicos (não usar root)
        2.Definir senhas fortes
        3.Usar GRANT e REVOKE para permissões
        4.Restringir acesso por IP
        5.Usar conexão segura (SSL)
        6.Manter o banco atualizado

15. Explique o conceito de buffer de memória no MySQL e sua importância para a performance do banco de dados.

    O buffer é uma área de memória RAM usada para:
        1.armazenar dados acessados com frequência
        2.reduzir acesso ao disco
        3.acelerar consultas

    Importância:
        1.melhora muito o desempenho
        3.evita lentidão em consultas repetidas
